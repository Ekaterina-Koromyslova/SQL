**Структура репозитория** 
1 папка = 1 проект
Внутри каждой папки есть подробное описание заданий. Код находится по пути src/ex_/day_ex_.sql
- Индивидуальные проекты имеют название DAY_
- Групповые проекты имют название TEAM_

**Описание проекта**
SQL Bootcamp — это комплексный учебный курс, в рамках которого последовательно изучаются ключевые аспекты работы с реляционными базами данных. В ходе выполнения заданий я прошла путь от базовых выборок и соединений таблиц до построения рекурсивных запросов, разработки триггеров и функций, моделирования графов и решения задач оптимизации, а также работы с архитектурой хранилищ данных (DWH). Каждое упражнение приближено к реальным кейсам аналитики, разработки бизнес-логики и работы с транзакционными системами.

Курс состоит из последовательных тематических дней, каждый из которых формирует новое направление компетенций: агрегирование данных, создание представлений, оптимизация через индексы, управление транзакциями и уровнями изоляции, построение ETL-логики, аудит изменений, вычисления внутри СУБД, работа с графами и маршрутизацией, анализ “грязных” данных и формирование отчётности.

**Использованные инструменты и технологии**
PostgreSQL — основная СУБД, на которой выполнены все лабораторные работы.

ANSI SQL — язык запросов, без использования проприетарных расширений вне специально разрешённых заданий.

psql — командная строка PostgreSQL, работа в нескольких параллельных сессиях.

CTE / WITH RECURSIVE — рекурсивные и нерекурсивные временные таблицы.

Views и Materialized Views — виртуальные и материализованные представления.

B-Tree Indexes — простые, составные, уникальные и частично уникальные индексы.

Триггеры и триггерные функции — аудит CRUD-операций, автоматизация изменений.

SQL Functions и PL/pgSQL — SQL-функции, параметризованные функции, функции-обёртки.

ETL-подходы — работа с ODS и DWH слоями, обработка данных из микросервисов.

Работа с аномалиями данных — пропущенные значения, несогласованные ключи, временные версии.

Моделирование графов в SQL — таблицы рёбер, рекурсивный обход, поиск маршрутов.

Аналитические запросы — группировки, вложенные агрегаты, вычисляемые поля.

Уровни изоляции транзакций — READ COMMITTED, REPEATABLE READ, SERIALIZABLE.

Диагностика производительности — EXPLAIN и EXPLAIN ANALYZE.

**Приобретённые навыки**
1. Моделирование данных и аналитика

построение сложных SQL-запросов, включая подзапросы, CTE, рекурсивные выражения;

анализ данных через агрегирование, группировки, вычисляемые поля и многотабличные объединения;

работа с временными диапазонами, пропущенными значениями и неполными данными.

2. Оптимизация производительности

создание и выбор оптимальных индексов;

работа с покрывающими индексами, частичными индексами и функциональными индексами;

чтение и анализ планов выполнения EXPLAIN ANALYZE;

практическое понимание, когда и как индексы ускоряют или замедляют запросы.

3. Рабочая логика и автоматизация внутри СУБД

создание триггеров для аудита и автоматической обработки изменений;

написание SQL и PL/pgSQL функций, в том числе с параметрами, значениями по умолчанию и управляющими конструкциями;

реализация бизнес-правил на уровне базы данных.

4. Транзакции и конкурентность

глубокое понимание уровней изоляции транзакций;

воспроизведение аномалий: Lost Update, Non-Repeatable Read, Phantom Read;

работа с параллельными сессиями, блокировками и deadlocks.

5. Работа с хранилищами данных (DWH)

понимание архитектуры ODS → DWH;

соединение данных из разрозненных систем (микросервисов);

извлечение исторических значений по временным признакам;

расчёт отчётных метрик (конвертация валют, суммы, агрегаты).

6. Решение задач оптимизации и графового анализа

моделирование графов с помощью SQL;

построение маршрутов, гамильтоновых циклов и реализация элементов задачи коммивояжёра;

работа с рекурсией для обхода графов.

7. Полный стек работы с SQL в реальных условиях

чтение, обработка и очистка “грязных” данных;

работа с OLTP и OLAP-подходами;

построение итоговых витрин данных;

понимание того, как SQL используется в бизнесе — от транзакционной логики до аналитики.
